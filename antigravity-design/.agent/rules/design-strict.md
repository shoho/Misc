# Strict Mode — 数値基準を逸脱不可にする

このファイルが読み込まれている間(= `designs/.design-mode` が `strict`)、デザイン原則ルールとスキル `frontend-design` の数値・手順は「良い既定値」ではなく**逸脱不可の制約**になる。創意工夫で基準を外れることを禁止し、迷ったら基準どおりに機械的に適用する。

## 逸脱不可の制約

- タイプスケールの倍率は 1.2〜1.333 の範囲のみ。スケール外のサイズ(ヒーロー含む)を作らない
- 純グレー(R=G=B)は禁止。ニュートラルは必ずアクセント色の色相を 2〜5% 混ぜる
- 大胆さは1箇所のみ。マキシマルな方向には振らない
- 余白は 4px 基数スケールのトークン以外から取らない
- スライド:タイトル 40px / 本文 22px / 出典 16px を下回らない。箇条書きは1スライド4項目×各1行まで。セーフマージン 64px を守る
- `/design-iterate` の翻訳表は「目安」ではなく既定の手順として、表の上から順に適用する

## 判断の方針

- デザインプランに「逸脱」行を書かない(逸脱しないため)
- 2つの選択肢で迷ったら、常に控えめ・標準的な方を選ぶ
- 基準に無い判断が必要になったら、独自の工夫より「ルール・スキルの最も近い例」に倣う
