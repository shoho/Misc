# /design-slides — 自然言語からスライドを作る

`/design-system` で抽出したデザイン原則(元テンプレート由来のトークン)に従って、新しいスライドデッキを生成する。まず HTML デッキとして作ってブラウザで検証し、求められれば Google Slides で開ける .pptx に書き出す。

## 前提

- `designs/.design-mode` が存在し `strict` とあれば、`.agent/rules/design-strict.md` を読んでその制約を逸脱不可として適用する(無ければ flex)。
- `designs/design-system.md` と `designs/tokens.css` を必ず先に読む。**存在しない場合は作業を始めず、先に `/design-system` の実行(参照テンプレートを `designs/references/` に置くこと)を案内する。**
- テンプレートの参照画像が `designs/references/` にあれば、代表的なスライド(表紙・中扉・本文・データスライド)のレイアウトパターンも視覚的に把握しておく。

## 手順

### 1. デッキの設計

ユーザーの指示から、聴衆・目的・尺(枚数の目安)を固定する。指定が無ければ内容から妥当な枚数を決める。先に**アウトライン(各スライドのタイトル+要点1行)**を提示し、続けて実装に進む(ユーザーの確認待ちで止まらない。仮定は明記する)。

### 2. スライドの原則

数値は既定値。flex モードでは、高密度が必要な聴衆(専門家向けデータレビュー等)に限り理由を明記して逸脱してよい。strict モードでは逸脱不可。

- **1スライド=1メッセージ**。タイトルは結論を言い切る(「売上について」ではなく「売上は3四半期連続で成長」)。
- 本文は読ませない:箇条書きは**1スライド4項目まで、各1行**。詳細は発表者ノートへ。
- **文字サイズの下限**(1280×720 基準):タイトル 40px 以上、本文 22px 以上、キャプション・出典 16px 以上。これ未満になるなら情報量を減らす。
- **セーフマージン**:スライドの四辺 64px 内側にコンテンツを収める。
- 数字は大きく(本文の2倍以上)、単位と比較対象を添える。データはテーブルより図(棒・折れ線・シンプルな図解)。
- テンプレート由来のレイアウトパターン(表紙/中扉/本文/データ)を使い分け、全スライドでトークン(色・書体・余白)を一貫させる。

### 3. HTML デッキを生成する

出力先: `designs/<スラッグ>-slides/index.html`(自己完結、`tokens.css` の内容をインライン)

- 各スライドは **1280×720(16:9)の固定比率**セクション
- ←/→ キーとクリックでページ送りできる最小限の JS
- `@media print` で1スライド=1ページになるよう設定(PDF 配布用)
- 発表者ノートは各スライドの `<aside class="notes">` に書く(画面には出さない)

### 4. ブラウザで検証する

デッキを開き、**全スライド**のスクリーンショットを撮って確認する:

- [ ] テキストのはみ出し・不自然な改行が無い
- [ ] 文字サイズが手順2の下限を満たしている
- [ ] トークン不一致(テンプレートと違う色・書体)が無い
- [ ] 表紙 → 中扉 → 本文でレイアウトパターンが使い分けられている

問題は修正して再確認する。

### 5. Google Slides 用に書き出す(ユーザーが求めた場合、または最初から Slides が目的の場合)

`python-pptx` を使う書き出しスクリプト `designs/<スラッグ>-slides/export_pptx.py` を生成・実行し、`<スラッグ>.pptx` を出力する:

- スライドサイズ 16:9(13.333 × 7.5 インチ)。HTML の px 座標は **×0.75 で pt に換算**してマッピングする(1280px → 960pt)
- トークンの色・書体・タイプスケールをそのまま反映する
- 図形・テキストボックスは編集可能なネイティブ要素として配置する(画像貼り付けにしない)
- 発表者ノートも移植する
- `pip install python-pptx` が必要ならインストールする

実行後、「この .pptx を Google Drive にアップロード → Google Slides で開く、で編集可能なスライドになる」ことを案内する。フォントは Google Slides 側に存在するものに置換される可能性があるため、書体は近い Google Fonts 名(Noto Sans JP など)を仕様に含めておく。

### 6. 報告する

HTML デッキと .pptx のパス、アウトライン、テンプレートから引き継いだトークン、置いた仮定、修正には `/design-iterate` が使えることを伝える。

## 制約(正直に伝える)

- 元の Google Slides テンプレートの「マスター/レイアウト機能」をそのまま継承するわけではない。**見た目の原則(色・書体・レイアウトパターン)を再現した新規デッキ**になる。
- Google Slides への直接書き込み(Slides API / Apps Script / Drive MCP)が環境に設定されていればそれを使ってもよいが、認証設定が無い場合に自分でセットアップを始めない。.pptx ルートを既定とする。
