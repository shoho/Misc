# /design-iterate — デザインを対話で磨く

`/design` で作ったプロトタイプ(または指定された HTML)を、ユーザーのフィードバックに沿って改良する。

## 手順

1. **対象を特定する**
   ユーザーが指定しなければ `designs/` 配下で最後に更新されたデザインを対象にする。対象ファイルと現状のスクリーンショットをまず確認する。

2. **フィードバックを変更項目に分解する**
   曖昧な要望(「もっと今風に」「重い感じがする」)は、具体的な変更(余白のスケール、書体ウェイト、彩度、階層)に翻訳してから着手する。解釈が複数ありうる場合は、採用した解釈を報告に明記する。

3. **トークン単位で変更する**
   色・余白・タイプスケールの変更は `:root` のカスタムプロパティを更新して全体に波及させる。個別要素へのその場しのぎの上書きでカスケードを汚さない。ライト/ダーク両テーマの整合を保つ。

4. **変更しない部分を守る**
   フィードバックに含まれない部分のデザイン判断は維持する。毎回全体を作り直さない。

5. **ブラウザで再検証する**
   変更後にスクリーンショットを撮り、ビフォー/アフターで意図どおりかを自分で確認してから報告する。

6. **報告する**
   「何を・なぜ・どう変えたか」を箇条書きで簡潔に。次に調整できそうな点があれば1〜2個だけ提案する。

## 補足

- 大きな方向転換(トーン自体の変更、レイアウト刷新)を求められたら、実装前に新しいデザインプランを短く提示してから進める。
- 反復の途中でユーザー指定と過去の自分の判断が矛盾したら、常にユーザー指定を優先する。
