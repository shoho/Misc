# /design-handoff — 実装へのハンドオフ

`designs/<スラッグ>/` のプロトタイプを、実装エージェント(Claude Code・Antigravity 自身・人間の開発者)にそのまま渡せる仕様一式にまとめる。

## 手順

1. **対象を特定する**
   指定が無ければ `designs/` 配下で最後に更新されたデザインを対象にする。

2. **`designs/<スラッグ>/HANDOFF.md` を生成する**
   次のセクションで構成する:
   - **概要**:何のための画面/ページか、対象ユーザー、達成すべき仕事(1段落)
   - **デザイントークン**:このデザインで使用した色・書体・スケールの一覧(`tokens.css` があればそこへの参照+差分)
   - **レイアウト構造**:セクション/コンポーネントの分解と、それぞれの責務・レスポンシブ挙動
   - **インタラクション仕様**:hover/focus/active、状態遷移、アニメーション(duration/easing 込み)、`prefers-reduced-motion` 時の挙動
   - **両テーマ**:ライト/ダークの差分ポイント
   - **アクセシビリティ要件**:フォーカス順、コントラスト、キーボード操作
   - **実装ノート**:プロトタイプ HTML のどこを見れば各部の実装が分かるか、プロトタイプと本実装で変えるべき点(データ結合、コンポーネント分割の提案)
   - **未決事項**:仮定のまま残っている判断

3. **アセットを揃える**
   最終スクリーンショット(両テーマ)を `designs/<スラッグ>/` に保存し、HANDOFF.md から参照する。

4. **報告する**
   バンドルの場所と、実装エージェントに渡すときのひとこと指示例(例:「designs/<スラッグ>/HANDOFF.md を読んで、この仕様どおりに実装して」)を示す。
